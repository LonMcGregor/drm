Types of DRM

Arbitrary check

An arbitrary check to determine whether to allow reading the file.

DVD region codes

DVDs may restrict playback to just one region of the world

Ineffective as they are easily bypassed by using a universal DVD reader

If a customer buys a DVD from abroad, they cannot play it at home

Server activation

A call to a server to determine whether to allow reading the file. If the server is unavailable, the file cannot be read.

Software

Performs a check when first starting the software to see if the user has legitimately purchased it

Ineffective as the code that performs the check can be bypassed, activation codes and logins can be shared

If a customer who legally purchased software wants to activate it, but the server no longer responds, they cannot use the software

Always online server check

Similar to server activation, but happens continuously, not just once

Games

While the game is played, repeatedly check that it has been legitimately purchased

Ineffective as the code that performs the check can be bypassed, activation codes and logins can be shared

If a player who legally purchased the game loses their internet connection (even temporarily), or the server no longer responds, they will stop being able to play the game

Encryption at rest

A legally purchased file that is unreadable without the accompanying license. After the license is also downloaded, the original file may continue to be encrypted and unreadable.

Ebooks, Adobe digital editions, Kindle ebooks

After the ebook is downloaded, it comes in the form of an encrypted file. It is unreadable until the unlocking license is downloaded.

Ineffective as the file needs to be decrypted to be read, and once it is decrypted, it can easily be copied, defeating the initial encryption

If a reader who legally purchases the book wants to copy the ebook for backup they are unable to do so as the files are encrypted

Case Study: Microsoft Ebook Store

Microsoft offered an ebook store. The books that were sold were encrypted and used DRM. Microsoft decided to close down their store one day, and all the customers who had bought books from them would find that their books were no longer readable. In this case Microsoft offered refunds, but no option was given to keep the ebooks after the store had closed.

https://www.bbc.co.uk/news/technology-47810367

Runtime obfuscation

When a program runs, but part of the code uses techniques commonly found in malware to evade investigation by the user

Games, Denuvo

The core logic of the game is hidden away and obfuscated, making it impossible for the player to investigate. May use virtual execution environments, obfuscated code pathways and obtuse design.

Ineffective as ultimately the code needs to run in order for the game to run, and it can be reverse engineered.

If a player who bought the game wanted to modify (“mod”) it to add new gameplay mechanics, they would be unable to do so

Proprietary reader

A digital file can only be read using a special program or website

Ebooks, Adobe digital editions, Kindle ebooks, Comixology

Once an ebook is purchased it can only be read by visiting the website it was from, or only using a device provided by a specific manufacturer. This has related issues of vendor lock in and vertical integration.

Ineffective as ultimately the file needs to be readable by a human, and the human could copy it that way

If a reader who legally purchased the book wanted to use their own reader, e.g. to customise the font for accessibility reasons, they would be prohibited from doing so.

Case Study: Comixology

Comixology provides a proprietary reader for the comics it sells, with a “guided view” technology. Users can download normal DRM-free comics, but only at the discretion of the publisher, so if the publisher doesn’t provide DRM-free versions, readers would be forced to use Comixology’s implementation.

Software trusted pathway

Operating systems may give applications an API which prevents 3rd party applications from accessing data within it

Streaming APIs

When playing a media file, the viewer is unable to make copies of the content as it plays

Ineffective as ultimately the file needs to be readable by a human, and the human could copy it that way

If a paying viewer wanted to take a screenshot of a tv show, to use it in a critical review, they would be unable to do so

Case Study: Netflix

The official Netflix apps for Windows 10 and Android both prevent the taking of screenshots. However, as this kind of protection isn’t available on all web browsers, if you view using Google Chrome or Firefox, you can take screenshots.

Case Study: Banking Apps

If you were using a mobile banking app you may want to protect your privacy by preventing screenshots from being made. Banking apps such as Nationwide’s implement this. However if you wanted to take a screenshot to keep in your own records, you are prevented from doing this.

https://developer.android.com/reference/android/view/WindowManager.LayoutParams#FLAG_SECURE

Hardware trusted pathway

A hardware implementation of DRM which provides an “authorised” route for data to flow, to prevent it being copied

HDMI / HDCP

A specification for HDMI screens and cables where any interruption or incompatibility results in the screen not displaying the content

Ineffective as it can be bypassed with hardware splitters

If a customer buys an older TV or HDMI cable, it may be unable to connect to newer peripherals, and they will no longer be able to use it
